GW-Builder v1.1.2-RC1 — Must-Pass Test Gates (for RC1 Closeout)
==============================================================

Legend:
- Gate: pass/fail hard requirement for RC1 acceptance.
- Evidence: command output, logs, screenshots, or captured files.

RC1 LAB TOPOLOGY GATE (T0)
--------------------------
Gate:
- 1x Admin Portal (GW-Admin Portal)
- 2x GW Nodes (gw-node-1, gw-node-2)
- Management VLAN reachable from portal to both GWs
- eth0 = public/underlay (non-trunk), eth1 = inside trunk (802.1Q enabled)

Evidence:
- ip link, ip -d link show
- /etc/gateway/gw.conf roles set consistently on both GWs
- portal /etc/hosts contains gw-node entries (gw-*)

SOURCE OF TRUTH GATES (T1)
--------------------------
T1.1 SoT init + node registration
Gate:
- gwreg init succeeds
- nodes added with mgmt_ip + vtep_ip
Evidence:
- gwreg node list

T1.2 Tenant allocation completeness
Gate:
- tenant alloc creates unique: tenant /25, inside /25, loopback /32, tunnel /30, VLAN, VNI, VRF, svc IPs
Evidence:
- gwreg tenant show <tenant>

T1.3 Manifest render + integrity
Gate:
- manifest JSON generated with registry_hash and correct node placement
Evidence:
- cat /etc/gateway/registry/tenants/<tenant>.json

TENANT APPLY + ISOLATION GATES (T2)
----------------------------------
T2.1 Deterministic manifest apply (both nodes)
Gate:
- gwctl tenant apply --manifest works on gw-node-1 and gw-node-2
Evidence:
- gwctl output logs + ip/vrf objects present

T2.2 Primary/Backup behavior
Gate:
- only PRIMARY activates loopback VIP and IPsec termination
- backup stages but does not terminate
Evidence:
- ip addr show lo (VIP present only on primary)
- ipsec statusall (conn up only on primary)
- frr/bgp status as applicable

T2.3 VRF isolation (no route bleed)
Gate:
- tenant A cannot reach tenant B networks
- default VRF cannot route into tenant VRF except via defined handoffs
Evidence:
- ping/traceroute negative tests
- ip rule / ip route show vrf <vrf>

VXLAN OVERLAY GATES (T3)
------------------------
T3.1 Per-tenant VXLAN device created on both nodes
Gate:
- vxlan<VNI> exists on both gw nodes
Evidence:
- ip -d link show | grep vxlan

T3.2 Static peer programming (RC1 mode)
Gate:
- VTEP peer list programmed and traffic passes across nodes
Evidence:
- bridge fdb show / ip neigh / tcpdump on vxlan device

T3.3 Cross-node tenant dataplane validation
Gate:
- a tenant-hosted service or test namespace on GW1 can reach GW2 tenant segment
Evidence:
- ping between tenant test endpoints across nodes

IPSEC + ROUTING GATES (T4)
--------------------------
T4.1 strongSwan up/down creates VTI and binds correctly
Gate:
- VTI appears with correct /30 addressing and VRF association

- ip link show vti* ; ip addr ; ip route show vrf

T4.2 Tenant BGP (FRR/zebra/bgp) session established (as configured)
Gate:

Evidence:
- vtysh show bgp summary ; show route vrf
