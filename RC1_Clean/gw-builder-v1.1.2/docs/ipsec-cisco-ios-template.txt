Cisco IOS/IOS-XE IKEv2 + VTI + BGP Template (Reference)
=======================================================

NOTE: This is a reference template. Adapt interface names, IPs, and crypto
to your standards. The goal is:
- IKEv2 PSK
- VTI tunnel interface
- BGP over the tunnel
- Routed subnets behind the peer

Assumptions
- Peer public IP: <PEER_WAN_IP>
- Gateway public IP: <GW_WAN_IP>
- Tunnel /30: 192.168.248.0/30
  - Cisco: 192.168.248.1/30
  - Gateway: 192.168.248.2/30
- BGP:
  - Cisco ASN: <CISCO_ASN>
  - Gateway ASN: <GW_ASN>

1) IKEv2 Proposal / Policy
crypto ikev2 proposal GW-PROP
 encryption aes-cbc-256
 integrity sha256
 group 14
!
crypto ikev2 policy 10
 proposal GW-PROP
!

2) Keyring / Profile (PSK)
crypto ikev2 keyring GW-KR
 peer GW
  address <GW_WAN_IP>
  pre-shared-key <PSK>
!
crypto ikev2 profile GW-IKEV2
 match identity remote address <GW_WAN_IP> 255.255.255.255
 authentication remote pre-share
 authentication local pre-share
 keyring local GW-KR
!

3) IPsec Transform / Profile
crypto ipsec transform-set GW-TS esp-aes 256 esp-sha256-hmac
 mode tunnel
!
crypto ipsec profile GW-IPSEC
 set transform-set GW-TS
 set ikev2-profile GW-IKEV2
!

4) VTI Tunnel Interface
interface Tunnel100
 ip address 192.168.248.1 255.255.255.252
 tunnel source <WAN_INTERFACE_OR_IP>
 tunnel destination <GW_WAN_IP>
 tunnel mode ipsec ipv4
 tunnel protection ipsec profile GW-IPSEC
!

5) Routing / BGP
router bgp <CISCO_ASN>
 neighbor 192.168.248.2 remote-as <GW_ASN>
 neighbor 192.168.248.2 description GW Tenant Tunnel
 !
 address-family ipv4
  neighbor 192.168.248.2 activate
  network <YOUR_INTERNAL_PREFIX> mask <MASK>
 exit-address-family

Operational checks
- show crypto ikev2 sa
- show crypto ipsec sa
- show interface tunnel100
- show bgp ipv4 unicast summary
