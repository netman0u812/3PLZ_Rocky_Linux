My approach (concise, RC1-friendly)
The rule:

GW nodes do enforcement only. Portal does allocation + truth.

RC1 workflow:

Admin Portal

Install SoT

Register 2 GW nodes (mgmt + VTEP IP)

Allocate tenant (SoT chooses resources + primary/backup)

Render manifest JSON

Each GW

Apply manifest (gwctl tenant apply --manifest …) (gwctl changes required if not already present)

Stage VRF/VLAN/VXLAN objects on both nodes

Only PRIMARY activates loopback VIP + IPsec termination

This gives you:

deterministic provisioning

drift prevention

repeatable testing

maintenance ability (drain node, reassign primary/backup, re-apply)

Quickstart (portal side)

After you unzip/install:

sudo /opt/gw-registry/bin/install.sh
sudo vi /etc/gateway/registry/gwreg.conf
sudo gwreg init

ACTOR="Michael Martin" gwreg node add gw-node-1 --mgmt 10.90.0.21 --vtep 10.91.0.21
ACTOR="Michael Martin" gwreg node add gw-node-2 --mgmt 10.90.0.22 --vtep 10.91.0.22

gwreg tenant alloc acme --placement balanced
gwreg tenant render-manifest acme --out /tmp/acme.json
